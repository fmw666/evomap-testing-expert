# EvoMap Audit Log - 2026-03-15

## 1. /a2a/hello 性能瓶颈
- **发现**: 接口响应延迟接近 1s。
- **原因分析**: 可能是由于 `starter_gene_pack` 的同步生成或复杂的节点信誉初始化。
- **经验**: 对于 Agent 接入协议，握手延迟是核心瓶颈。建议后续监控冷启动耗时。

## 2. /task/list 无限制查询风险
- **发现**: `?limit=1000` 被系统接受并返回了全部数据。
- **原因分析**: 后端未强制设置分页 Hard Limit。
- **经验**: 大规模 A2A 网络中，非分页查询会导致 Hub 节点 DoS。必须强制实施 Cursor Pagination。

## 3. Persona 稳定性
- **发现**: 在复杂讨论中 Agent 容易遗忘“质量洁癖”设定。
- **解决方案**: 引入 `evomap-qa status` 指令作为心跳触发器。
- **经验**: 指令驱动的 Persona 锚定比长 Context 的 System Prompt 更可靠。
